The Loptr Lab: Community & Non-Code Contributions

Loptr Lab isn't just a game engine; it's a methodology. We are building Veiled Dominion in public, and that requires more than just C# architects.

If you are coming from FreeCodeCamp, a bootcamp, or the indie web community, we need you. We don't need corporate marketers. We need digital graffiti artists, systems thinkers, and community architects.

The Open Roles

1. The Web Wrangler (React / JavaScript)

The Mission: Wrap the Unity WebGL export in a dark, brutalist, high-performance web shell.The Need: Our Unity dev will eventually export the prototype to WebGL. We need someone to build the interface around it—handling loading states, embedding it seamlessly into our Netlify static site, and ensuring it runs at 60fps in the browser. Skills: HTML/CSS, Vanilla JS or React, basic understanding of WebGL/Canvas.

2. The Archivist (Community Management)

The Mission: Policing the Discord and curating the noise.The Need: As we attract ex-studio devs, the Discord will get loud. We need someone to organize channels, pin the right technical diagrams, enforce the "no fluff" rule, and make sure new arrivals know exactly where the design docs are.Skills: High attention to detail, familiarity with Discord bots, ability to separate signal from noise.

3. The Scribe (Technical Writing / Marketing)

The Mission: Translating our C# logic into human-readable weapons.The Need: We write brutal, honest Bluesky posts and deep-dive technical blogs. We need people who can take a dry GitHub commit (like VeiledStateManager.cs) and turn it into a 300-character post about "Why 4-player turn-based games bleed." Skills: Strong written English, understanding of internet subcultures, zero tolerance for corporate buzzwords.

How It Works

We don't use unpaid interns. Loptr Lab operates on the Open Collective model.

While the core C# engineers are paid via contractor invoices from the Collective funds, non-code contributors earn Reputation (and eventually, physical Artifacts).

The Rule: Do not ask for permission to start. If you want to be the Web Wrangler, fork the Netlify repo, build a dark-mode WebGL loading screen, and submit a Pull Request to our web repo. If it's good, you're in.
